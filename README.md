# How to use the Github repo for program development
When you want to create/implement something new into the program **create a new branch** specifically for implementing and developing this new functionallity.
> This isolates your changes from the main branch and also prevents changes to main from messing up your code

Commit all your changes made when developing the new feature/function to this new branch, **NOT THE MAIN BRANCH**.

When you're ready to merge your changes into the main branch (e.g. feature finished) **create a pull request** so that (preferably someone other than you) can review the changes made and merge your branch and all its changes into the main branch.

If changes are made to the main branch during development, you can move these changes over to your development branch by using a pull request then going through and fixing any merge conflicts.
